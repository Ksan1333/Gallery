# Gallery 1.4.0 リグレッションテスト計画

## 1. 目的

1.4.0 で変更したギャラリースクロール終端、X ダウンロード候補・日時、GIF 変換メニューを確認し、一覧、ビューア、保存、設定の主要導線に回帰がないことを確認する。

実施結果は [regression_test_cases.tsv](regression_test_cases.tsv) に記録する。未実装の自動化候補は [automated_test_backlog.md](automated_test_backlog.md) で管理する。

## 2. 対象

- 画像、GIF、動画を含む 100 件以上のギャラリーと、縦横比が混在する Staggered Grid
- ジェスチャーナビゲーションと 3 ボタンナビゲーション
- X 投稿 URL（画像のみ、動画のみ、GIF、動画と投稿サムネイルが混在）
- MediaStore の日時列を確認できる端末または検証用 ContentResolver
- 静止画、GIF、MP4 を開いたメディア／動画ビューア

## 3. 優先度

| 優先度 | 基準 |
| --- | --- |
| P0 | データ保存、終了位置、クラッシュ、主要操作が不能になる問題 |
| P1 | 代表端末・代表データで正しさを確認する問題 |
| P2 | 境界条件、視認性、長時間運用の確認 |

## 4. 実施順

1. `:app:compileDebugKotlin` と `:app:testDebugUnitTest` を実行する。
2. TSV の P0 を実機で実施する。下端ドラッグはレーン長が不揃いなデータを必ず使う。
3. X ダウンロード後に Gallery と端末の詳細情報で日時・保存種別を確認する。
4. P1/P2 を実施し、端末、API レベル、結果、障害リンクを TSV に記録する。

## 5. 合格基準

- P0 がすべて PASS であること。
- スクロールバーを下端で離した時点で `canScrollForward` が false となり、一覧コンテンツの下端とバー下端が一致すること。
- 動画投稿の保存候補に投稿サムネイルが混入しないこと。
- 保存した画像・動画・GIF の日時がダウンロード日時と整合すること。
- 静止画・GIF で GIF 変換が実行不能、動画で実行可能であること。
