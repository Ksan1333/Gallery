# Gallery リグレッション試験書

## 1. 目的

Gallery 0.4.0 の主要動作を、リリース前や大きな改修後に再確認するための回帰試験書です。メディア一覧、ビューア、My List / AI 分析、フォルダ管理、X / Twitter ダウンロード、漫画ビューア、お絵描き資料、履歴・おすすめ、共通基盤までを対象にします。

実施結果を書き込みながら使うファイルは、タブ区切りの実施シートです。

- [リグレッション試験実施シート TSV](regression_test_cases.tsv)

自動実行できる回帰確認として、端末不要のローカル JUnit テストも用意しています。まずは X / Twitter ダウンロードの URL・保存形式判定、Room 変換、特徴ベクトル等価性など、壊れると影響が大きく、かつ端末に依存しないロジックを対象にしています。

```powershell
.\gradlew.bat :app:testDebugUnitTest
```

## 2. 使い方

`regression_test_cases.tsv` を Excel、Google スプレッドシート、LibreOffice Calc などで開き、以下の列に実施結果を記録します。

| 列 | 記入内容 |
| --- | --- |
| 実施結果 | `PASS`、`FAIL`、`保留`、`対象外` のいずれか |
| 実際結果/証跡 | 実際の表示、保存先、ログ、スクリーンショット名など |
| 不具合ID | 起票したチケット番号やメモ番号 |
| 実施日 | `YYYY-MM-DD` |
| 実施者 | 実施者名 |
| 備考 | 再現条件、端末差分、補足など |

## 3. 優先度

| 優先度 | 使いどころ |
| --- | --- |
| P0 | 毎回確認するスモーク・主要導線。落ちるとアプリ利用に大きく影響する項目。 |
| P1 | リリース前に確認する標準回帰。保存、編集、永続化、外部連携を含む項目。 |
| P2 | 大量データ、端末差分、ネットワーク異常、性能、長時間処理などの拡張確認。 |

## 4. 推奨試験環境

| 観点 | 推奨条件 |
| --- | --- |
| Android | API 24 以上。権限差分確認用に Android 13 以上を 1 台含める。 |
| メディア | JPG / PNG / WebP / GIF / MP4 を含む。大量表示確認用に 500 件以上の画像セットを用意する。 |
| フォルダ | DCIM 配下に複数フォルダ、空フォルダ、長いフォルダ名を用意する。 |
| 漫画 | ZIP と PDF をそれぞれ 1 件以上。ページ数が多いファイルも 1 件用意する。 |
| X / Twitter | 動画投稿、画像投稿、GIF 投稿、無効 URL、直接メディア URL を用意する。 |
| AI | AI 未分析画像、タグ済み画像、動画を混在させる。動画は分析対象外になることを確認する。 |
| ネットワーク | 通常回線、低速または一時切断、完全オフラインを確認できる状態。 |

## 5. 実施順序

1. インストール、権限、起動、ナビゲーションを確認する。
2. メディア一覧とビューアの基本動作を確認する。
3. My List、タグ、AI 分析、フォルダ、ゴミ箱、一括編集を確認する。
4. X / Twitter ダウンロード、漫画ビューア、お絵描き資料など外部連携やファイル操作を確認する。
5. おすすめ、視聴履歴、共通基盤、性能、異常系を確認する。
6. 開発環境では `.\gradlew.bat :app:testDebugUnitTest` を実行し、ローカル自動テストも確認する。

## 6. 合格基準

- P0 がすべて `PASS`、または既知の制限として合意済みであること。
- P1 の `FAIL` がリリース判断上許容できる範囲に整理されていること。
- P2 の未実施や保留は、対象端末やデータ条件と一緒に記録されていること。
- クラッシュ、ANR、データ消失、誤削除、メディア形式の誤保存は原則リリースブロッカーとして扱うこと。

## 7. 重点回帰観点

- 28 列表示時のスクロール応答性と ANR。
- ビューアのタップ、ダブルタップ、ピンチ、パン、左右スワイプの競合。
- 一覧からビューアへ開いた項目と、戻った後の表示位置の一致。
- 長押し選択とドラッグ範囲選択のズレ。
- X / Twitter 共有 URL がクリップボード URL より優先されること。
- X / Twitter の GIF 投稿が GIF として保存されること。
- AI 分析の進捗、キャンセル、温度制御、動画除外。
- MediaStore、Room、アプリ内ゴミ箱、完全削除の整合性。
