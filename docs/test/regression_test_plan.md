# Gallery 0.6.0 リグレッションテスト計画

## 1. 目的

0.6.0で追加・変更した類似画像グループ、フォルダグループ、テーマコントラスト、X投稿の複数メディア保存を確認し、既存の一覧、ビューア、フォルダ移動、ゴミ箱、設定、本の主要導線に回帰がないことを確認する。

実施項目は[regression_test_cases.tsv](regression_test_cases.tsv)へ記録する。

## 2. 対象

- メディア一覧、2/3/4/7/28列、スクロール、選択
- 類似画像グループと表示設定
- フォルダ一覧、表示グループ、メディア移動
- カスタムテーマ、Materialメニュー、ダイアログ
- X / Twitterの単一・複数動画/GIF保存
- メディア/動画/ブックビューア
- ゴミ箱、設定バックアップ、検索履歴

## 3. 優先度

| 優先度 | 基準 |
| --- | --- |
| P0 | クラッシュ、ANR、データ消失、保存形式違い、主要導線停止につながる項目 |
| P1 | リリース前に確認する表示・永続化・操作回帰 |
| P2 | 大量データ、端末差、通信失敗などの拡張確認 |

## 4. 推奨環境

| 観点 | 条件 |
| --- | --- |
| Android | API 31を必須に含め、API 29以下とAPI 33以上を各1台 |
| メディア | JPG/PNG/WebP/GIF/MP4、特徴ベクトル生成済み画像、500件以上の一覧 |
| 類似画像 | 隣接類似度60%ちょうど、60%未満、32件以上の連鎖データ |
| フォルダ | 4フォルダ以上、空フォルダ、同名に近い名称、カスタムサムネイル |
| X投稿 | 動画1件、動画2件以上、GIF2件以上、動画とGIF混在、無効URL |
| テーマ | 全プリセット、文字色と背景色を同色にしたカスタムパレット |

## 5. 自動テスト

```powershell
./gradlew.bat :app:testDebugUnitTest
./gradlew.bat :app:compileDebugKotlin
```

自動化対象:

- 60%境界と隣接連鎖、件数上限なし、吹き出し順とビューア対象リスト
- フォルダグループ保存データの往復変換、不正値除外、表示順、重複所属除外
- 複数メディアの画質候補集約、GIF変換要否
- テーマの文字コントラストとMaterialコンテナ色
- URL判定、GIF/MP4形式判定、既存の変換・共通ロジック

## 6. 実施順

1. 自動テストとKotlinコンパイル
2. P0スモーク
3. 類似画像・列数切り替え
4. フォルダグループとフォルダ移動
5. テーマ全プリセットとカスタム同色設定
6. X単一・複数保存
7. ビューア、ゴミ箱、本、設定バックアップ
8. P1/P2、長時間スクロール、通信異常

## 7. 合格基準

- P0がすべてPASS
- P1のFAILにデータ消失、クラッシュ、ANRがない
- 自動テストとコンパイルが成功
- 既知制限は再現条件と回避策を記録
- 保存件数、ファイル形式、ファイル名、履歴件数が投稿内メディア数と一致
