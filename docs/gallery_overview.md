# Gallery アプリ概要書

## 1. 目的

Gallery は、標準ギャラリーアプリでは足りない高度な分類、検索、AI分析、フォルダ管理、視聴履歴を備えた個人向けメディア管理アプリである。

写真、GIF、動画、漫画ファイルを大量に扱う前提で、閲覧の速さ、戻り位置の維持、詳細なフィルタ、AIタグ付け、関連画像推薦をひとつのアプリ内で完結させることを目的とする。

## 2. 対象メディア

- 画像
- GIF
- 動画
- ZIP 形式の漫画
- PDF 形式の漫画
- X / Twitter から取得した動画・画像

## 3. 主な利用シーン

- 端末内の写真、GIF、動画をまとめて一覧する
- 年、月、日単位で大量メディアを俯瞰する
- AIでタグ、年齢制限、画像ベクトルを分析する
- タグ、未整理、未分析、お気に入りなどの条件でメディアを探す
- 画像・GIF・動画を全画面ビューアで閲覧する
- 視聴履歴をもとにおすすめ画像を表示する
- フォルダ単位で整理し、フォルダの表示順やサムネイルを管理する
- 不要なメディアをゴミ箱へ移動し、復元または完全削除する
- 漫画ファイルをページ単位で閲覧する
- X / Twitter の共有URLや貼り付けURLから動画・画像を保存する

## 4. 機能一覧

### 4.1. メディア一覧

- 端末内の画像、GIF、動画を MediaStore から取得する
- Room DB にメタデータ、タグ、分析結果、削除状態を保存する
- Paging と LazyVerticalGrid によって大量メディアを表示する
- ピンチ操作で列数を 1、3、4、7、28 に切り替える
- 列数に応じて日、月、年単位のグルーピングへ自動切り替えする
- 日付ヘッダーを表示する
- 高速スクロールバーで任意位置へ移動する
- スクロール時にトップバーを隠し、表示領域を広げる
- 28列表示では年単位の俯瞰を優先し、1年あたり最大6段の表示に制限する
- 28列表示では小さいサムネイルを範囲制限付きで読み込み、スクロール負荷を抑える
- 動画には再生時間ラベル、GIFには GIF ラベルを表示する
- お気に入り状態をグリッド上に表示する

### 4.2. フィルタ・ソート

- メディア種別で絞り込む
  - すべて
  - 画像
  - 動画
  - GIF
- AI判定の年齢制限で絞り込む
  - すべて
  - SFW
  - R15
  - R18
- 端末向けアスペクト比で絞り込む
  - すべて
  - スマホ向け
  - PC向け
- 日付、サイズ、名前で並び替える
- 昇順、降順を切り替える

### 4.3. 選択・一括編集

- 長押しで選択モードに入る
- 長押しドラッグで範囲選択する
- 選択済みメディアを起点にドラッグした場合は範囲選択解除する
- 選択数が0件になった場合は通常モードへ戻る
- 一括お気に入りを実行する
- 一括ゴミ箱移動を実行する
- 一括タグ追加を実行する
- 一括年齢制限変更を実行する
- 一括フォルダ移動を実行する
- フォルダ画面では選択した1件をフォルダサムネイルに設定できる

### 4.4. メディアビューア

- 画像、GIF、動画を全画面表示する
- タップでシステムバーとアプリバーの表示・非表示を切り替える
- ダブルタップでズームイン・ズームアウトする
- ピンチで拡大縮小する
- 拡大中はパン操作できる
- 左右スワイプで前後のメディアへ移動する
- 上スワイプでレコメンドタブを表示する
- 下スワイプでビューアを閉じる
- 通常ギャラリーから開いた場合、ビューアで最後に表示したメディアを戻り位置にする
- レコメンドから開いた場合、戻り時に対象メディアがギャラリー中央に来るようにする
- お気に入り切り替え、タグ編集、削除、復元、完全削除を実行する
- 画像、動画フレーム、GIFフレームを壁紙に設定する
- 動画フレーム、GIFフレームをスクリーンショットとして保存する
- フォルダサムネイルに設定する
- ファイル名、パス、日時、サイズ、解像度、タグ、AI判定などの詳細情報を表示する

### 4.5. 動画ビューア

- Media3 ExoPlayer で動画を再生する
- 再生、一時停止、ミュートを切り替える
- 送り戻し間隔を 1、2、5、10、30、60 秒から選択する
- 10秒相当の戻し・送り操作を提供する
- シークバーで再生位置を移動する
- 画面下部の操作領域でフレーム単位のコマ送りを行う
- 動画フレームを取得して保存できる

### 4.6. GIFビューア

- Coil の GIF 対応で GIF を再生する
- GIFフレーム抽出を行う
- フレーム単位のステップ表示を切り替える
- 表示中フレームを画像として保存する
- 表示中フレームを壁紙に設定する

### 4.7. マイリスト

- お気に入り一覧を表示する
- 未整理メディアを表示する
- AI未分析メディアを表示する
- ベクトル未分析メディアを表示する
- タグ別カテゴリを表示する
- タグ件数、代表サムネイルを表示する
- カテゴリの長押しドラッグで並び替える
- カテゴリ選択モードで複数カテゴリを扱う
- AI分析対象期間を選択する
  - 7日
  - 30日
  - すべて
  - カスタム日数
- AI分析の見積もり時間を表示する

### 4.8. フォルダ管理

- 端末内フォルダをスキャンする
- 管理対象フォルダを登録する
- フォルダ別にメディアを表示する
- フォルダの表示順を変更する
- フォルダのカスタムサムネイルを設定・リセットする
- DCIM 配下に新規フォルダを作成する
- フォルダ選択画面からメディア移動先を指定する

### 4.9. ゴミ箱

- メディアをアプリ内ゴミ箱へ移動する
- ゴミ箱内メディアを一覧表示する
- ゴミ箱から復元する
- 完全削除する
- 複数選択して復元または完全削除する
- ゴミ箱ビューアでは通常削除ではなく復元・完全削除操作を表示する

### 4.10. AI分析

- Smiling Wolf / WD Tagger 系 ONNX モデルで画像タグを推定する
- 推定タグを DB に保存する
- タグ信頼度を保存する
- rating タグとキーワード判定から SFW / R15 / R18 を推定する
- MediaPipe Image Embedder で画像ベクトルを生成する
- 画像ベクトルを DB に保存する
- タグ類似度とベクトル類似度で関連画像を抽出する
- 未分析のみを対象にして重複分析を避ける
- 期間指定により分析対象を絞り込む
- モデル未配置時は ModelDownloader で必要ファイルを取得する
- 分析は foreground service とグローバル進捗で実行する
- キャンセル状態を管理する
- ONNX Runtime は XNNPACK、NNAPI を試行し、失敗時はCPU実行へフォールバックする
- MediaPipe は GPU を試行し、失敗時は CPU へフォールバックする

### 4.11. レコメンド・視聴計測

- 計測モードを切り替える
- メディア閲覧回数と閲覧時間を保存する
- 閲覧実績をもとにおすすめメディアを表示する
- ビューア内のレコメンドタブに関連メディアを表示する
- ベクトル類似、タグ類似、ランダム候補を組み合わせて推薦する

### 4.12. 漫画機能

- ZIP と PDF の漫画ファイルを端末ストレージからスキャンする
- フォルダ別に漫画ファイルを表示する
- ZIP 内画像、PDF ページをサムネイル化する
- 漫画ビューアでページ単位に閲覧する
- 縦画面では単ページ表示、横画面では見開き表示に対応する
- 右開き相当の reverse layout に対応する
- タップ領域で前後ページ移動とUI表示切り替えを行う
- ピンチズームとパンに対応する
- ページスライダーとプレビューを表示する
- 表示ページをスクリーンショットとして保存する
- 画面回転を切り替える
- Android 11 以降では全ファイルアクセス権限を案内する

### 4.13. X / Twitter ダウンロード

- 共有インテントから X / Twitter のURLを受け取る
- x.com / twitter.com の status URL を開いた場合にダウンロード画面へ遷移する
- クリップボード内URLを検出する
- 手入力URLに対応する
- fxtwitter API から動画・画像候補URLを解決する
- 直接メディアURLにも対応する
- High / Medium / Low の画質選択を行う
- 動画は Movies/Gallery、画像は Pictures/Gallery に保存する
- ダウンロード履歴を保存する
- ダウンロード中、完了、失敗の状態を表示する
- 履歴を消去する
- ダウンロード後にギャラリーを更新する

### 4.14. サムネイル生成

- アプリ起動後にサムネイル生成サービスを開始する
- 画像サムネイルをキャッシュファイルとして生成する
- 動画は MediaMetadataRetriever で代表フレームを取得する
- DB にサムネイル有無を保存する
- 生成はバックグラウンドで実行する
- グローバル進捗とキャンセル状態を管理する

## 5. 技術構成

| --- | --- |
| 言語 | Kotlin |
| UI | Jetpack Compose, Material Design |
| 画面遷移 | Navigation Compose |
| 一覧表示 | LazyVerticalGrid, Paging Compose |
| ローカルDB | Room SQLite |

| 画像読み込み | Coil |

## 6. パフォーマンス方針

- UIスレッドで重いソート、フィルタ、AI分析、サムネイル生成を行わない
- 10,000件以上のメディアでも一覧スクロールが破綻しないことを目標にする
- 28列表示ではスクロール中の新規サムネイル読み込みを抑制し、キャッシュ済み画像と軽量セルで俯瞰性を保つ
- 年単位表示では1年あたり最大6段に制限し、表示件数を抑える
- Coil のメモリキャッシュ、ディスクキャッシュを利用する
- AI分析は未分析メディアを中心に順次処理する
- 分析中は進捗表示とキャンセルを提供し、ユーザー操作への影響を抑える
