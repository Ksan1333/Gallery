- Paging の未ロードセルを 0 高さで破棄せず、実セルサイズの軽量プレースホルダーとして描画するように変更し、スクロールバーの最終ジャンプ後に `visibleItems` が大量発生する問題を軽減しました。
- スクロールバーのドラッグ中もグリッド本体が追従するように、`requestScrollToItem()` を一定間隔で実行するライブ追従を追加しました。
- 28列表示では Paging 表示を使わず、1年あたり6段までに制限した軽量リストを表示するように変更しました。
- 28列表示でも小さいサムネイルを表示するように戻し、スクロール中はキャッシュ済み画像のみ描画して操作のもたつきを抑えるようにしました。
- 28列表示への切り替え時に、古い全件グリッド生成結果が後から反映されて軽量リストを上書きする問題を防止しました。
- 28列表示への切り替え直後、軽量リストが完成する前に古い全件リストを描画・プリロードしないようにしました。
- ホームギャラリーのスクロール位置を保存し、アプリを開き直した時やバックグラウンド復帰時も最後に見ていた位置へ復元するようにしました。
- ギャラリーのスクロールバー操作中に、バーの左側へ現在位置の年月日ラベルを表示するようにしました。
- バックグラウンド復帰後にギャラリーが意図しない位置へ移動する原因を追えるよう、`GALLERY_SCROLL_RESTORE_TRACE` ログを追加しました。
- ギャラリーのスクロールバーを一番下までドラッグした時に、末尾まで到達するように位置計算を修正しました。
- ホームギャラリーのスクロール位置復元をアプリ実行中の一時状態に限定し、アプリ終了後の次回起動では前回位置へ戻さないようにしました。
- スクロールバー操作中に Paging の総アイテム数が増えた場合でも、指を離した時点の最新件数で末尾へ追従するようにしました。
- バックグラウンド復帰後のギャラリー位置復元を、index だけでなく表示中メディアの URI を基準に再アンカーするようにし、Paging の総数変化による位置ずれを抑制しました。
- ギャラリー位置保存時に Paging の一時的な空スナップショットを参照してクラッシュする問題を修正しました。
- Paging 側が未ロードで `peek()` から URI を取得できない場合も full grid 側へフォールバックし、BG/FG 復帰時の URI アンカー保存が null になり続ける問題を修正しました。
- BG/FG 復帰中に古い復元処理の完了通知が新しい復元処理の pending 状態を解除し、途中位置を保存してしまう問題を修正しました。
- 詳細設計書のUIモック10枚を、実装画面のトップバー、一覧、ダイアログ、ボトムナビ、操作ボタンが伝わる精密なSVGに更新しました。
- 詳細設計書のUIモックを実装画面に合わせて再作成し、黒ヘッダー、DarkGray操作バー、実際のボトムナビ文言、CategoryScreen、各TopAppBar、Creators/お気に入りサイトの茶系テーマを反映しました。
- 詳細設計書のUIモックを機能内の画面単位に分割し、文字で代用していたメニュー、追加、削除、書き出し/読み込み、しおり、外部表示などのアイコンをSVG線画アイコンへ置き換えました。
- 各詳細設計書に機能内画面遷移図を追加し、画面タイトル付きのミニUIモック、Dialog、Popup、BottomSheet、遷移矢印で機能単位の流れを確認できるようにしました。
